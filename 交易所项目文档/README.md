3. [交易所项目-术语表.md](交易所项目-术语表.md)：统一“订单/成交/账本/清算/行情”等口径，避免跨模块歧义。
4. [交易所项目-核心功能矩阵.md](交易所项目-核心功能矩阵.md)：把“该有的核心功能”一次性列全（实现为先、可开关）。
5. [交易所项目-架构总览.md](交易所项目-架构总览.md)：整体架构、边界与关键链路（用于统一认知）。

9. [交易所开发-通用总纲.md](交易所开发-通用总纲.md)：按模块细化实现要点（尽量不放“决策”，决策写 ADR）。

## 开发顺序（MVP 推荐）
1. 决策先行：完成 [交易所项目-落地文档.md](交易所项目-落地文档.md) 的“项目快照/范围勾选”，拍板 ADR-001/002/003/007（从 [ADR/ADR-000-索引.md](ADR/ADR-000-索引.md) 进入）
2. 契约先行：冻结 [交易所项目-接口契约.md](交易所项目-接口契约.md) + [交易所项目-数据模型.md](交易所项目-数据模型.md) 的最小集（能联调/能写验收用例）
3. 交易闭环：下单/撤单 -> 撮合 -> 清算 -> 账本/余额 -> 私有回报
4. 行情闭环：深度/逐笔/订阅 + 断线回补（先快照重拉，再做增量回补）
5. 上线门槛：按 [交易所项目-测试与验收.md](交易所项目-测试与验收.md) 压测与演练，通过后再开放入口
